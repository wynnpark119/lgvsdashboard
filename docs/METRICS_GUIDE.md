# LG VS Dashboard 연산 지표 가이드

> 이 문서는 대시보드에서 사용되는 **연산 지표**의 산출식과 **왜 이렇게 설계했는지**를 설명합니다.

---

## 1. 통합 퍼널 (Integrated Funnel)

### 1.1 왜 통합 퍼널인가?

**기존 문제:**
- LG.com 방문자만 집계하면 전체 마케팅 활동의 일부만 보임
- LinkedIn에 예산을 많이 투입하는데, 그 성과가 퍼널에 반영되지 않음
- 채널별로 따로 보면 전체 파이프라인 건강도를 판단하기 어려움

**해결:**
- LinkedIn + LG.com + YouTube를 **하나의 통합 퍼널**로 합산
- 채널별 **가중치**를 적용해 "관심의 질"을 반영
- 모든 채널을 하나의 숫자로 볼 수 있어 의사결정이 명확해짐

---

### 1.2 채널별 가중치

**위치:** Home → 주목 현황 카드 → TOFU/MOFU/BOFU 수치

| 채널 | 가중치 | 왜 이 가중치인가 |
|------|--------|-----------------|
| **LG.com** | 1.0 | Owned 채널, 직접 방문 = 능동적 관심 표현. 전환(문의)이 발생하는 곳 |
| **LinkedIn** | 0.6 | 주력 채널, 예산 비중 높음. 하지만 피드 노출은 LG.com 방문보다 관심 깊이가 낮음 |
| **YouTube** | 0.4 | 콘텐츠 소비 채널. 영상 시청은 관심 표현이지만, 전환 기여도는 상대적으로 낮음 |

**왜 단순 합산이 아닌 가중치인가:**
- LinkedIn 조회 1회 ≠ LG.com 방문 1회 (관심의 질이 다름)
- LinkedIn 피드에서 스쳐 지나간 조회와, LG.com에 직접 검색해서 들어온 방문은 가치가 다름
- 예산 비중(LinkedIn이 높음)과 관심 깊이(LG.com이 높음)의 균형 필요

---

## 2. TOFU (Top of Funnel) 산출

### 2.1 산출식

**위치:** Home → 주목 현황 카드 → TOFU 수치

```
TOFU = LG.com 첫방문 × 1.0 
     + (LinkedIn 조회수 ÷ 10) × 0.6 
     + (YouTube 조회수 ÷ 10) × 0.4
```

### 2.2 왜 이렇게 계산하는가

| 요소 | 설명 | 왜 이렇게 |
|------|------|----------|
| **LG.com × 1.0** | 기준값 | 직접 방문은 가장 강한 관심 표현. Owned 채널이므로 기준(1.0)으로 설정 |
| **LinkedIn ÷ 10** | 스케일 조정 | LinkedIn 조회수는 LG.com 방문보다 10배 이상 많음. 스케일을 맞춰야 비교 가능 |
| **× 0.6** | 가중치 | 예산 비중이 높지만, 피드 노출은 능동적 방문보다 관심 깊이가 낮음 |
| **YouTube ÷ 10** | 스케일 조정 | YouTube 조회도 LG.com보다 많음. 스케일 조정 |
| **× 0.4** | 가중치 | 영상 시청은 관심 표현이지만, 전환 기여도가 LinkedIn보다 낮음 |

### 2.3 계산 예시

```
LG.com 첫방문: 12,847
LinkedIn 조회: 285,000 → ÷10 = 28,500 × 0.6 = 17,100
YouTube 조회: 42,000 → ÷10 = 4,200 × 0.4 = 1,680
─────────────────────────────────────────────────────
TOFU = 12,847 + 17,100 + 1,680 = 31,627
```

## 3. MOFU (Middle of Funnel) 산출

### 3.1 산출식

**위치:** Home → 주목 현황 카드 → MOFU 수치

```
MOFU = LG.com MOFU × 1.0 
     + LinkedIn Engagement × 0.6 
     + (YouTube 50%+ 시청 ÷ 10) × 0.4
```

### 3.2 왜 이렇게 계산하는가

| 요소 | 설명 | 왜 이렇게 |
|------|------|----------|
| **LG.com MOFU** | 재방문 + 체류 2분+ + 영상 50%+ | 단순 방문이 아닌 "심화 관심" 표현. 여러 조건 충족 필요 |
| **LinkedIn Engagement** | 좋아요 + 댓글 + 공유 | 단순 조회가 아닌 "상호작용". 관심을 적극 표현한 것 |
| **YouTube 50%+ 시청** | 영상 절반 이상 시청 | 끝까지 보지 않아도 50% 이상 = 관심 있게 본 것 |

**왜 TOFU와 다른 지표를 쓰는가:**
- TOFU = "첫 접촉" (노출, 첫 방문)
- MOFU = "심화 관심" (상호작용, 재방문, 깊은 소비)
- 같은 채널이라도 TOFU/MOFU에서 측정하는 행동이 다름

## 4. BOFU (Bottom of Funnel) 산출

### 4.1 산출식

**위치:** Home → 주목 현황 카드 → BOFU 수치

```
BOFU = LG.com 문의 폼 제출 건수
```

### 4.2 왜 이렇게 계산하는가

| 요소 | 설명 | 왜 이렇게 |
|------|------|----------|
| **문의만 집계** | 스펙 요청, 데모 요청 없음 | 현재 LG VS에서 수집 가능한 전환 지표는 "문의"뿐 |
| **LG.com에서만** | 다른 채널에서 문의 없음 | LinkedIn/YouTube에서는 문의 폼이 없음. LG.com이 유일한 전환 지점 |

**왜 단순한가:**
- BOFU는 "실제 비즈니스 기회"를 측정
- 복잡한 가중치보다 **실제 문의 건수**가 가장 명확한 지표
- 향후 스펙 요청, 데모 요청 등이 추가되면 확장 가능

## 5. 퍼널 전환율

### 5.1 산출식

**위치:** Home → 주목 현황 카드 → 퍼널 사이 화살표

```
TOFU→MOFU 전환율 = (MOFU 수 / TOFU 수) × 100
MOFU→BOFU 전환율 = (BOFU 수 / MOFU 수) × 100
```

### 5.2 왜 이렇게 계산하는가

| 요소 | 왜 이렇게 |
|------|----------|
| **비율로 표시** | 절대 수치보다 "얼마나 다음 단계로 넘어가는가"가 파이프라인 건강도를 더 잘 보여줌 |
| **단순 나눗셈** | 복잡한 가중치 없이, 누구나 이해할 수 있는 직관적 계산 |

### 5.3 해석 기준

| 전환율 | 해석 | 왜 이 기준 |
|--------|------|----------|
| 20%+ | 건강한 파이프라인 | B2B 마케팅에서 상위 퍼포먼스 |
| 10~20% | 일반적 수준 | 업계 평균 |
| ~10% | 전환 병목 | 콘텐츠/전략 점검 필요 |

## 6. 검토 강도 (Review Intensity)

### 6.1 산출식

**위치:** 기술별 관심 현황 → 기술 카드 → "검토 강도: 6.6/10"

```
Review Intensity (1-10) = Σ(정규화된 입력값 × 가중치) × 10
```

### 6.2 입력 지표 및 가중치

| 입력 | 가중치 | 정규화 | 왜 이 가중치 |
|------|--------|--------|-------------|
| 세션 수 | 15% | min(sessions / 5000, 1) | 관심 볼륨. 중요하지만 양보다 질이 더 중요 |
| 평균 체류 시간 | 20% | min(avgTime / 5분, 1) | 콘텐츠 몰입도. 오래 머무름 = 깊은 관심 |
| 영상 시청 깊이 | 20% | viewDepth / 100 | 이해 동반 여부. 끝까지 봄 = 진지한 검토 |
| 재방문율 | 20% | returnVisitRate / 100 | 지속 관심. 다시 옴 = 계속 검토 중 |
| 복수 콘텐츠율 | 15% | multiContentRate / 100 | 심화 탐색. 여러 콘텐츠 봄 = 비교/검토 |
| Inquiry 수 | 10% | min(inquiryCount / 100, 1) | 구체적 관심. 문의 = 실제 관심 표현 |

### 6.3 왜 이렇게 계산하는가

**문제:**
- 세션 수만 보면 "피상적 관심"과 "깊은 검토"를 구분할 수 없음
- 체류 시간만 보면 "방치"와 "몰입"을 구분할 수 없음

**해결:**
- 여러 지표를 **종합**해서 "진짜 검토 중인가"를 판단
- 각 지표의 **가중치**로 "무엇이 더 중요한가"를 반영
- 1-10 스케일로 **직관적 비교** 가능

### 6.4 계산 예시 (Digital Cockpit)

```
세션 4500 → 4500/5000 = 0.90 × 0.15 = 0.135
체류 3.8분 → 3.8/5 = 0.76 × 0.20 = 0.152
영상 72% → 0.72 × 0.20 = 0.144
재방문 45% → 0.45 × 0.20 = 0.090
복수콘텐츠 38% → 0.38 × 0.15 = 0.057
Inquiry 85 → 85/100 = 0.85 × 0.10 = 0.085
─────────────────────────────────────────
합계 = 0.663 × 10 = 6.6
```

## 7. 단계 분류 (TOFU/MOFU/BOFU)

### 7.1 산출식

**위치:** 기술 카드 상단 라벨

```
IF Intensity ≥ 7.0 → BOFU
IF 4.0 ≤ Intensity < 7.0 → MOFU
IF Intensity < 4.0 → TOFU
```

### 7.2 왜 이 기준인가

| 기준 | 왜 이 값 |
|------|----------|
| **7.0 (BOFU)** | 여러 지표가 모두 높아야 도달 가능. "문의 발생" 수준의 관심 |
| **4.0 (MOFU)** | 일부 지표가 높음. "심화 탐색 시작" 수준 |
| **4.0 미만** | 아직 첫 접촉 수준 |

## 8. 광고 의존도 (Paid Dependency)

### 8.1 산출식

**위치:** 주목 현황 카드 → 각 단계의 "Paid X%"

```
Paid Dependency = Paid 트래픽 / 전체 트래픽 × 100
```

### 8.2 분류 기준

| Paid 비율 | 분류 | 왜 이 기준 |
|----------|------|----------|
| ~25% | Low | 3/4 이상이 Organic = 자생적 관심 |
| 25~50% | Medium | 균형 상태, 모니터링 필요 |
| 50%+ | High | 과반이 광고 = 광고 중단 시 급감 위험 |

### 8.3 왜 이 지표가 필요한가

**문제:**
- 광고로 트래픽을 올리면 숫자는 좋아 보이지만, 광고 중단 시 급감
- "자생적 관심"과 "인위적 관심"을 구분해야 지속 가능성 판단 가능

**해결:**
- Paid 비율을 명시적으로 표시
- High일 경우 "광고 의존 주의" 경고

## 9. 종합 신호 (Signal Type)

### 9.1 결정 로직

**위치:** 기술별 검토 현황 테이블 → "신호" 컬럼

```
IF (Stage = BOFU) AND (Paid = Low) AND (Direction ≠ 하락)
  → 🟢 Green: "문의 전환 · Engagement 高"

IF (Stage = MOFU) AND (Direction = 유지)
  → 🟡 Yellow: "추가 콘텐츠 노출"

IF (Paid = High)
  → 🟠 Orange: "광고 의존 주의"

IF (Direction = 하락)
  → 🔴 Red: "Engagement 하락"

ELSE
  → ⚪ Gray: "모니터링"
```

### 9.2 왜 이렇게 계산하는가

**문제:**
- 여러 지표(단계, 방향, 광고 의존도)를 개별로 보면 판단이 어려움
- 임원/의사결정자는 "한눈에" 상태를 파악하고 싶음

**해결:**
- 여러 조건을 조합해 **하나의 색깔/레이블**로 표시
- 색깔만 봐도 "어떤 기술에 주목해야 하는지" 즉시 파악

## 10. 캠페인 영향 분류 (Campaign Influence)

### 10.1 산출식

**위치:** 캠페인 영향 분석 → 캠페인 카드

```
Retention = (종료 2주 후 Intensity - 시작 전 Intensity) / (Peak - 시작 전) × 100
Lift = (Peak - 시작 전) / 시작 전 × 100
```

### 10.2 분류 기준

| 분류 | 조건 | 왜 이 기준 |
|------|------|----------|
| **Amplifier** | Retention ≥ 60% AND Lift ≥ 20% | 캠페인 효과가 지속됨. 좋은 투자 |
| **Neutral** | 중간 | 보통 효과 |
| **Distorter** | Retention < 30% OR Lift < 10% | 단기 반응만. 효과 재검토 필요 |

### 10.3 왜 이렇게 계산하는가

**문제:**
- 캠페인 성과를 노출/클릭으로만 보면 "일시적 스파이크"도 성공으로 보임
- 캠페인 종료 후 효과가 유지되는지가 더 중요

**해결:**
- **Retention**: 캠페인 종료 후에도 효과가 유지되는가?
- **Lift**: 캠페인으로 얼마나 상승했는가?
- 둘 다 높아야 "진짜 효과 있는 캠페인"
